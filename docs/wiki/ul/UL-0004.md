## 현재결정사항

AI 요약(구조화·한줄요약·개인화 매칭) 2차 가공을 AWS Lambda 공용 프록시 대신 각 사용자의 **Claude 루틴**(Claude Code 자체 스케줄 실행)으로 옮기는 방안을 검토한 결과, **현재는 도입하지 않는다.**

1. **도입 보류**: Claude 루틴을 AI 가공 경로로 코드·인프라에 반영하지 않는다. 사유: (a) 분산 fork 구조에서 운영자가 관찰·개입 가능한 지점은 현재 Lambda 프록시가 유일하며, 이를 없애면 프로젝트가 "운영 불가능한 5개 독립 프로덕션"이 된다. (b) 도입이 해결하려는 문제(Gemini 레이트리밋 몰림)가 5명·1개월 규모에서는 발생 가능성이 낮아 확실한 도입 비용(온보딩 마찰, 환경 편차, 관측성 상실)으로 불확실한 이득을 사는 거래로 판단.
2. **opt-in 텔레메트리로 관측성 대체 불가**: 각 사용자 루틴이 상태를 리포팅하는 방안은 "신호 없음"이 실패인지 정상 유휴인지 구분 불가능한 한계로 기각. 제대로 하려면 중앙 감시 인프라를 다시 지어야 해 실익 없음.
3. **어댑터 공존(Lambda + Claude 루틴 동시 지원)도 채택하지 않음**: 실사용자가 사실상 한 경로뿐인 현재 규모에서 두 번째 경로는 검증 트래픽 없는 유지비용만 지는 죽은 추상화로 판단. 단일 경로(Lambda) 유지.
4. **README 레시피 문단은 허용**: "Claude 구독자는 프록시 대신 Git 소스에 직접 루틴을 붙일 수도 있다"는 안내는 코드 결합이 없어 완전히 가역적이므로 문서로만 언급 허용한다. 단, 이 문단에는 1차 소스 Git 파일 스키마가 **호환성 비보장(best-effort)**임을 반드시 명시한다.
5. **재검토 조건 명시**: 향후 사용자 규모 확대로 운영자 부담 Gemini 비용·거버넌스가 실제 문제가 될 때 재검토한다. 이때도 도입 명분은 "레이트리밋 회피"가 아니라 "AI 비용·소유권의 사용자 분산"이어야 한다.
6. **Claude 루틴 채택 여부와 무관하게 지금 반영하는 조치 2가지**: (a) 1차 소스 Git 파일의 **스키마 버전 고정** — 저장소가 Notion에서 Git 파일로 전환되며 이 파일 포맷이 분산된 각 fork에 대한 사실상의 API가 되었으므로 필요. (b) 공고 레코드에 **안정적 고유 ID 부여**(예: 공고 URL 해시) — dedup·재처리·향후 파이프라인 단계 분리 전부의 전제가 되는 저비용 조치.
7. **Pending 남김**: 레이트리밋 대응 인프라(UL-0002의 토큰버킷·Reserved Concurrency 등)를 지터+지수 백오프 중심으로 경량화할지 여부는 이번 논의에서 확정하지 않고 추후 별도로 다룬다.

## Basis

UL-0002는 "사용자 fork에 AI 관련 시크릿을 요구하지 않는 것"을 목표로 Lambda 공용 프록시를 도입했다. 이번 UL은 그 프록시 구조를 대체할 대안(Claude 루틴)을 검토했으나 채택하지 않기로 하여, UL-0002의 결정을 뒤집지 않고 오히려 재확인한다 — 상충이 아니다.

다만 이번 결정은 UL-0002/UL-0003에 없던 새로운 근거를 하나 추가한다: **Lambda 프록시가 단순히 "시크릿 부담을 없애는 수단"을 넘어, 분산 fork 구조에서 운영자가 유일하게 관찰·개입할 수 있는 지점(운영 시임)이라는 점**이다. 이 근거는 향후 이 프록시를 다른 방식으로 대체하려는 논의가 재발할 경우, "시크릿 부담 제거"뿐 아니라 "운영 시임 보존" 여부도 함께 검토해야 함을 의미한다.

스키마 버전 고정과 레코드 고유 ID 부여는 Claude 루틴 채택 여부와 무관하게 필요하다고 판단한 조치로, UL-0002에서 저장소를 Notion에서 Git 파일로 전환하기로 한 결정의 자연스러운 후속 조치다(파일 포맷이 사실상의 API가 되었다는 사실은 당시 명시적으로 다루지 않았던 함의).

레이트리밋 대응 인프라 경량화 여부는 이번 논의에서 제기되었으나 확정하지 않고 Pending으로 남긴다 — UL-0002의 토큰버킷·지터·Reserved Concurrency 조합이 5명 규모에 과설계일 수 있다는 지적이 있었으나, 이를 확정하려면 별도 검토가 필요하다.
